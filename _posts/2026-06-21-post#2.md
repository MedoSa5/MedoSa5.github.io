Phase 0 is completed, with two remaining issues still unresolved.

On the infrastructure side, I set up Rocky Linux 9 and Ubuntu Server locally using VMware. Rocky required two iterations: I initially deployed version 10, then later realized it does not match the RHCSA/RHEL 9 exam baseline. I rebuilt the environment accordingly to stay aligned with the certification scope.

My GitHub profile is now active, the README is in place, and this blog is live with its first post documenting the direction of the project.

Two blockers remain on the external services side. Oracle Cloud and AWS both failed payment verification with my RedotPay virtual card. This appears to be a known limitation with fintech-issued cards and verification systems. As a result, Phase A will be executed entirely on local virtual machines, which remains consistent with the roadmap since cloud usage is not required at this stage.

Separately, Stripe and alternative services (Revolut, Wise, Payoneer, MyFin) are not available from Algeria under current conditions. There is no immediate workaround, so this is recorded as a constraint to be addressed before Phase C rather than a current blocker.
